## 如何检查最新部署

### 方法 1：在 Vercel 控制台检查

1. 刷新 Vercel 控制台的 "Deployments" 页面
2. 查看最新的部署：
   - 如果看到新的部署（比 20 分钟前更新的），说明正在部署
   - 查看部署的 commit hash，应该包含 `1680b03` 或 `906efaa`

### 方法 2：查看部署日志

1. 点击最新的部署（最上面的那个）
2. 查看 "Build Logs" 或 "Function Logs"
3. 检查是否有错误信息

### 方法 3：直接测试

访问以下 URL 测试：
- **主页**：`https://aio-2-git-master-bupoos-projects.vercel.app`
- **API 健康检查**：`https://aio-2-git-master-bupoos-projects.vercel.app/api/health`

如果 API 返回包含 `path` 和 `url` 字段的 JSON，说明最新修复已生效。

## 如果还没有新部署

Vercel 通常会在几分钟内自动检测到 GitHub 的更新。如果超过 5 分钟还没有新部署：

1. 在 Vercel 控制台点击 "Redeploy" 按钮
2. 或者手动触发部署：
   - 进入项目设置
   - 找到 "Git" 设置
   - 点击 "Redeploy" 或 "Trigger Deployment"
